# VoiceDraw AI 纯语音绘图助手设计文档

## 一、项目背景

本项目对应题目二“AI 语音绘图工具”，目标是开发一款纯语音控制的绘图工具。用户在完成浏览器麦克风授权后，不需要通过鼠标或键盘进行绘图创作，而是通过语音指令完成图形绘制、组合图案生成、复杂指令执行、撤销、清空和保存图片等操作。

题目要求重点关注以下几个方面：

1. 指令理解的准确性与容错性；
2. 语音到绘图操作的响应延迟；
3. 复杂指令的拆解与执行能力；
4. 在实现应用的同时，额外提交一份设计文档，记录计划支持哪些指令能力，最终实现了哪些，以及未完成部分的原因说明。

基于以上要求，本项目采用前端网页方式实现，使用 HTML、CSS、JavaScript、Canvas API 和浏览器 Web Speech API 构建一个轻量级 AI 语音绘图原型系统。

---

## 二、项目目标

本项目希望实现一个简单、直观、低成本的语音绘图工具，使用户可以通过自然语言语音指令完成绘图操作。

项目主要目标包括：

1. 支持用户通过语音输入绘图指令；
2. 支持浏览器将语音转换为文本；
3. 支持从文本指令中解析图形类型、颜色、位置和操作意图；
4. 支持绘制基础图形和组合图案；
5. 支持生成一幅完整的随机风景画；
6. 支持复杂语音指令的拆解和顺序执行；
7. 支持撤销、清空画布、保存图片等画布操作；
8. 显示指令解析与 Canvas 绘图执行耗时，用于体现响应延迟；
9. 保持系统轻量化，尽量不依赖后端服务和额外付费接口。

---

## 三、纯语音控制说明

由于浏览器安全策略限制，网页首次调用麦克风时必须由用户主动点击按钮并授权。因此，本项目页面保留了“启动语音控制”和“停止语音控制”按钮。

需要说明的是：

- 启动按钮仅用于触发浏览器麦克风授权；
- 麦克风授权完成后，绘图创作过程通过语音完成；
- 用户可以通过语音完成绘图、清空、撤销、保存和停止语音识别；
- 项目的核心交互方式是语音控制，而不是键盘输入或鼠标绘制。

## 四、计划支持的指令能力

项目设计初期，计划支持以下几类语音指令能力。

### 1. 基础图形绘制指令

计划支持用户通过语音绘制基础图形，例如：

- 画一个圆形；
- 画一个矩形；
- 画一个三角形。

### 2. 颜色控制指令

计划支持用户通过语音指定图形颜色，例如：

- 红色圆形；
- 蓝色矩形；
- 黄色三角形；
- 绿色图形。

### 3. 位置控制指令

计划支持用户通过语音指定图形位置，例如：

- 在左边画一个圆形；
- 在右边画一个矩形；
- 在中间画一个三角形；
- 在上方画一个太阳；
- 在下方画一个图形。

### 4. 组合图案绘制指令

计划支持一些由多个 Canvas 图形组合而成的图案，例如：

- 太阳；
- 房子；
- 云朵；
- 星星；
- 笑脸。

### 5. 场景生成指令

计划支持用户通过一句语音生成完整画面，例如：

- 画一幅风景画；
- 生成一幅完整画；
- 画一个场景。

### 6. 画布操作指令

计划支持用户通过语音完成画布管理操作，例如：

- 清空画布；
- 撤销；
- 返回上一步；
- 保存图片；
- 停止语音控制；
- 帮助。

### 7. 复杂指令拆解能力

计划支持用户一次说出包含多个步骤的复杂指令，例如：

- 清空画布，然后画一幅风景画，再在左边画一个太阳；
- 画一个太阳，然后在右边画一朵云；
- 画一幅风景画，接着在上方画一颗星星。

系统需要能够识别连接词，并将复杂指令拆解为多条简单指令后依次执行。

## 五、最终实现的功能

经过开发，本项目最终实现了以下功能。

### 1. 纯语音控制绘图

用户点击“启动语音控制”完成麦克风授权后，可以直接说出绘图指令。系统会调用浏览器 Web Speech API 进行语音识别，并将识别结果传入指令解析模块。

已实现示例：

- 在左边画一个红色圆形；
- 在右边画一个蓝色矩形；
- 在中间画一个黄色三角形；
- 画一个太阳；
- 在右边画一朵云；
- 在中间画一座房子。

### 2. 基础图形绘制

已实现以下基础图形：

- 圆形；
- 矩形；
- 三角形。

系统会根据用户指令中的关键词判断需要绘制的图形类型。

### 3. 颜色识别

已实现以下颜色识别：

- 红色；
- 蓝色；
- 绿色；
- 黑色；
- 黄色；
- 紫色；
- 橙色；
- 粉色。

例如，用户说“在左边画一个红色圆形”，系统会识别“红色”为颜色，“圆形”为图形，并在左侧绘制红色圆形。

### 4. 位置识别

已实现以下位置识别：

- 左边；
- 右边；
- 中间；
- 上方；
- 下方。

例如，用户说“在右边画一个蓝色矩形”，系统会将绘图位置映射到画布右侧区域。

### 5. 组合图案绘制

已实现以下组合图案：

- 太阳；
- 房子；
- 云朵；
- 星星；
- 笑脸。

这些图案不是单一图形，而是由多个 Canvas 基础绘图操作组合生成。例如，太阳由圆形和光线组成，房子由矩形、三角形、门和窗户组成。

### 6. 随机风景画生成

已实现语音生成完整风景画功能。

用户可以说：

- 画一幅风景画；
- 生成一幅完整画；
- 画一个场景。

系统会自动绘制包含多个元素的风景画，例如：

- 天空；
- 草地；
- 太阳；
- 云朵；
- 远山；
- 房子；
- 树木；
- 花朵；
- 小路；
- 河流等自然元素。

其中部分元素会带有随机变化，例如天空颜色、草地颜色、太阳位置、云朵数量、花朵颜色和位置等，使每次生成的画面略有不同。

### 7. 复杂语音指令拆解

项目已经实现复杂指令拆解能力。

例如用户说：

```text
清空画布，然后画一幅风景画，再在左边画一个太阳，再在右边画一朵云
```

系统会拆解为：

```text
1. 清空画布
2. 画一幅风景画
3. 在左边画一个太阳
4. 在右边画一朵云
```

然后按顺序执行每一步。

当前支持的拆解连接词包括：

- 然后；
- 再；
- 接着；
- 并且；
- 同时；
- 中文逗号；
- 中文句号；
- 中文分号。

该功能用于体现复杂语音指令的拆解与执行能力。

### 8. 画布操作功能

已实现以下语音操作：

- 清空画布；
- 撤销；
- 返回上一步；
- 保存图片；
- 停止语音控制；
- 帮助。

其中，“撤销”和“返回上一步”都可以触发撤销功能；“停止语音”“停止识别”“停止语音控制”等表达都可以停止语音识别。

### 9. 响应耗时显示

项目已新增指令执行耗时显示功能。

系统会在语音识别完成后，记录从文本指令解析开始到 Canvas 绘图执行完成之间的耗时，并在页面中显示类似以下内容：

```text
指令解析与绘图执行耗时：xx ms
```

需要说明的是，该时间主要表示“语音识别完成后，文本指令解析到 Canvas 绘图完成”的执行耗时。由于语音识别本身由浏览器 Web Speech API 完成，整体语音交互延迟仍会受到浏览器语音识别速度、网络环境、麦克风质量和用户发音清晰度的影响。

## 六、指令理解的准确性与容错性设计

为了提升指令理解的准确性，本项目采用关键词匹配的方式解析用户语音指令。

### 1. 图形关键词识别

系统会识别以下图形关键词：

- 圆形、圆；
- 矩形、方形、方块；
- 三角形、三角；
- 太阳；
- 房子、房屋；
- 云、云朵；
- 星星、五角星；
- 笑脸、表情。

### 2. 颜色关键词识别

系统会识别以下颜色关键词：

- 红；
- 蓝；
- 绿；
- 黑；
- 黄；
- 紫；
- 橙；
- 粉。

### 3. 位置关键词识别

系统会识别以下位置关键词：

- 左；
- 右；
- 中；
- 中间；
- 上；
- 下。

### 4. 操作关键词识别

系统会识别以下操作关键词：

- 清空；
- 撤销；
- 返回上一步；
- 保存；
- 停止语音；
- 停止识别；
- 停止语音控制；
- 结束语音；
- 帮助。

### 5. 容错性设计

为了提高容错性，系统没有只匹配唯一表达，而是为同一操作设置了多个相近表达。

| 用户表达 | 系统理解 |
|---|---|
| 矩形 / 方形 / 方块 | 矩形 |
| 云 / 云朵 | 云朵 |
| 房子 / 房屋 | 房子 |
| 撤销 / 返回上一步 | 撤销 |
| 停止语音 / 停止识别 / 停止语音控制 / 结束语音 | 停止语音控制 |
| 风景 / 一幅画 / 场景 / 完整画 | 生成完整风景画 |

这种方式虽然不如大语言模型理解能力强，但实现简单、响应快、成本低，适合作为前端原型系统。

## 七、语音到绘图操作的响应延迟设计

本项目的整体流程为：

```text
用户语音输入
↓
浏览器 Web Speech API 语音识别
↓
生成文本指令
↓
JavaScript 本地解析指令
↓
Canvas API 绘制图形
↓
页面显示执行结果和耗时
```

为了降低响应延迟，本项目采用了以下设计：

1. 不依赖后端服务器；
2. 不调用远程大语言模型接口；
3. 指令解析逻辑在浏览器本地完成；
4. Canvas 绘图操作直接在前端执行；
5. 使用关键词匹配方式快速判断用户意图；
6. 在页面中显示指令解析与绘图执行耗时。

因此，语音识别完成后，系统可以较快地完成文本指令解析和绘图操作。

需要说明的是，页面显示的耗时并不是用户从开始说话到图形出现的完整总耗时，而是语音识别完成后，从文本指令解析到 Canvas 绘图执行完成的耗时。完整语音交互延迟还会受到浏览器语音识别速度、网络环境和麦克风识别效果影响。

## 八、复杂指令拆解与执行设计

复杂指令拆解是本项目的重要功能之一。

用户可能不会每次只说一个简单命令，而是一次说出多个操作。例如：

```text
清空画布，然后画一幅风景画，再在左边画一个太阳，再在右边画一朵云
```

系统处理方式如下：

1. 获取完整语音识别文本；
2. 根据“然后、再、接着、并且、同时、逗号、句号、分号”等连接词拆分句子；
3. 去除空白内容；
4. 得到多个子指令；
5. 按顺序调用单条指令执行函数；
6. 每条子指令分别完成解析和绘图操作。

示例拆解结果：

```text
原始指令：
清空画布，然后画一幅风景画，再在左边画一个太阳，再在右边画一朵云

拆解后：
1. 清空画布
2. 画一幅风景画
3. 在左边画一个太阳
4. 在右边画一朵云
```

该设计使系统具备一定的复杂语音指令处理能力，能够完成连续绘图和多步骤画布操作。

## 十、未完成部分及原因说明

虽然本项目已经实现了纯语音绘图的核心功能，但由于开发时间、技术复杂度和成本限制，仍有一些功能暂未完成。

### 1. 未实现任意复杂自然语言理解

目前系统主要依靠关键词匹配解析指令，因此更适合处理结构较清晰的语音命令，例如“在左边画一个红色圆形”。

对于更加复杂、模糊的自然语言，例如“帮我画一个温暖一点的画面”或“把刚刚那个图形变大一点”，当前系统还不能完全理解。

原因：

- 这类功能需要更复杂的语义理解；
- 可能需要接入大语言模型；
- 会增加接口成本和系统复杂度；
- 当前项目更侧重低成本前端原型实现。

### 2. 未实现对象级精细编辑

目前系统支持撤销上一步，但不支持对某一个已经绘制的对象进行单独编辑。

例如暂未支持：

- 把左边的圆形变大；
- 删除刚刚画的云；
- 把房子移动到右边；
- 改变某个图形的颜色。

原因：

- Canvas 绘图默认是像素级绘制；
- 若要实现对象级编辑，需要为每个图形建立对象数据结构；
- 需要记录每个对象的位置、颜色、大小、类型和绘制顺序；
- 开发复杂度较高，超出当前原型阶段范围。

### 3. 随机风景画的构图仍有优化空间

目前系统可以生成随机风景画，但随机元素之间仍可能出现局部布局不够理想的情况。例如，部分自然元素的位置可能与房子、道路或其他元素距离较近，画面构图还不够智能。

原因：

- 当前随机风景画主要使用固定区域加随机偏移的方式生成；
- 尚未实现元素碰撞检测；
- 尚未实现画面美学评分；
- 尚未实现基于语义或布局规则的智能排版。

后续可以通过区域约束、碰撞检测和更精细的构图规则进一步优化。

### 4. 未实现完全免点击启动

由于浏览器安全策略限制，网页首次调用麦克风必须由用户主动点击并授权。因此项目无法做到页面打开后自动开始监听语音。

原因：

- 这是浏览器对麦克风权限的安全限制；
- 网页不能在没有用户操作的情况下自动调用麦克风；
- 因此项目保留“启动语音控制”按钮用于授权。

### 5. 未接入真实大语言模型

当前项目没有调用 GPT、DeepSeek 等大语言模型接口，而是使用本地关键词规则实现指令解析。

原因：

- 降低开发成本；
- 避免接口申请和调用费用；
- 减少网络请求带来的延迟；
- 保证项目可以直接在浏览器中运行；
- 更适合课程/比赛原型展示。

## 十一、后续优化方向

后续如果继续完善，可以从以下方向优化：

1. 接入大语言模型，提高自然语言理解能力；
2. 增加更多图形和场景，例如人物、动物、城市、海边、夜空等；
3. 增加对象级编辑能力，例如移动、删除、缩放、改色；
4. 为随机风景画增加元素碰撞检测，减少不合理重叠；
5. 增加语音指令日志，展示复杂指令拆解结果；
6. 增加更多容错表达，例如“帮我画”“来一个”“生成一个”等；
7. 优化语音识别失败时的提示信息；
8. 支持导出更多格式，例如 JPG、SVG；
9. 支持保存绘图历史记录；
10. 支持更多语言或中英文混合指令。

## 十二、项目总结

VoiceDraw AI 纯语音绘图助手实现了一个基于浏览器的轻量级语音绘图原型系统。用户在完成麦克风授权后，可以通过语音完成基础图形绘制、组合图案绘制、随机风景画生成、复杂指令拆解、撤销、清空和保存图片等操作。

项目重点围绕题目要求进行了设计：

1. 在指令理解方面，系统支持图形、颜色、位置和操作关键词识别；
2. 在容错性方面，系统为同一操作设置了多种相近表达；
3. 在响应延迟方面，系统采用本地 JavaScript 解析和 Canvas 绘图，并显示指令解析与绘图执行耗时；
