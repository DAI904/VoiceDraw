# VoiceDraw AI 语音绘图助手设计文档

## 一、项目背景

本项目选择题目二“AI 语音绘图工具”。题目要求开发一款纯语音控制的绘图工具，用户不能使用鼠标或键盘完成绘图操作，而是通过语音指令完成图形绘制。
考虑到开发周期较短，本项目采用轻量级 Web 应用形式实现原型系统，重点验证“语音输入 → 指令解析 → Canvas 绘图”的核心交互流程。

## 二、用户故事

### 用户故事 1：通过语音绘制基础图形

作为一名普通用户，我希望可以直接说出“在左边画一个红色圆形”，系统能够识别我的语音指令，并在画布对应位置绘制图形。

### 用户故事 2：通过语音控制图形颜色和位置

作为用户，我希望可以通过语音指定图形的颜色和位置，例如“在右边画一个蓝色矩形”，从而减少手动操作。

### 用户故事 3：通过指令管理画布

作为用户，我希望可以通过“撤销”“清空画布”等指令管理绘图过程，提高操作容错性。

## 三、计划支持的指令能力

项目原计划支持以下指令能力：
1. 基础图形绘制：圆形、矩形
2. 颜色识别：红色、蓝色、绿色、黑色、黄色、紫色
3. 位置识别：左边、右边、中间、上方、下方
4. 操作指令：撤销、清空画布
5. 语音输入：通过浏览器麦克风识别用户语音
6. 自动执行：识别语音内容后自动完成绘图操作

## 四、最终实现的功能

目前项目已实现以下功能：
1. 支持文字指令输入
2. 支持语音识别输入
3. 支持绘制圆形和矩形
4. 支持识别红色、蓝色、绿色、黑色、黄色、紫色
5. 支持识别左边、右边、中间、上方、下方位置
6. 支持撤销上一步
7. 支持清空画布
8. 支持浏览器 Canvas 绘图展示

## 五、核心实现思路

本项目采用 HTML、CSS 和 JavaScript 开发。
HTML 负责页面结构，包括输入框、按钮和画布区域；CSS 负责页面样式；JavaScript 负责语音识别、指令解析和 Canvas 绘图逻辑。
系统流程如下：
1. 用户点击“开始语音识别”按钮
2. 浏览器调用 Web Speech API 获取语音内容
3. 系统将语音转换为文字
4. JavaScript 对文字进行关键词解析
5. 根据关键词判断颜色、图形和位置
6. 调用 Canvas API 在画布上绘制对应图形

## 六、指令解析逻辑

项目当前采用关键词规则解析方式。
例如用户输入：
“在左边画一个红色圆形”
系统会解析出：
- “左” → 位置为 left
- “红” → 颜色为 red
- “圆” → 图形为 circle
然后调用画圆函数，在左侧坐标位置绘制红色圆形。

## 七、成本控制策略

本项目采用浏览器原生能力实现，不依赖后端服务器、数据库或付费大模型接口。
主要成本控制方式包括：
1. 使用前端静态页面，无需服务器部署
2. 使用浏览器 Web Speech API 完成语音识别
3. 使用关键词规则解析指令，避免调用大模型接口
4. 使用 Canvas 完成绘图，降低系统复杂度
5. 所有功能均可在本地浏览器运行

## 八、未完成功能及原因

由于开发周期较短，目前暂未实现以下功能：
1. 复杂自然语言理解  
   目前主要通过关键词规则解析指令，暂不支持复杂语义推理。
2. 多轮连续修改  
   当前支持撤销和清空，但暂不支持“把刚才的圆形变大”“移动到右边”等复杂编辑。
3. 更多图形类型  
   当前只实现圆形和矩形，后续可以扩展三角形、线条、箭头等图形。
4. 跨浏览器完全兼容  
   语音识别功能依赖浏览器支持，建议使用 Chrome 或 Edge 测试。

## 九、后续优化方向

后续可以从以下方向继续优化：
1. 接入大语言模型，提高复杂指令理解能力
2. 增加更多绘图元素，例如线条、三角形、文字等
3. 支持多轮对话式修改图形
4. 增加保存图片功能
5. 优化页面交互和视觉效果
6. 提升语音识别失败时的容错提示

## 十、总结

VoiceDraw 是一个基于语音指令的轻量级 AI 绘图工具原型。项目实现了语音识别、指令解析、Canvas 绘图、撤销和清空等核心功能，能够较完整地展示语音控制绘图的基本流程。
