# VoiceDraw AI 纯语音绘图助手设计文档

## 一、项目背景

本项目选择题目二“AI 语音绘图工具”。

题目要求开发一款纯语音控制的绘图工具，用户不能使用鼠标或键盘完成绘图创作，而是通过语音指令完成绘图操作。同时，题目要求综合考虑指令理解的准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令的拆解与执行能力。

考虑到开发周期较短，本项目采用轻量级 Web 应用形式实现原型系统，重点验证“语音输入 → 指令拆解 → 指令解析 → Canvas 绘图”的核心交互流程。

## 二、项目目标

本项目希望实现一个可以通过语音完成基础绘图创作的网页工具，核心目标包括：

1. 用户可以通过语音发出绘图指令。
2. 系统能够识别语音内容并转换为文本。
3. 系统能够从语音文本中解析出图形、颜色、位置和操作类型。
4. 系统能够支持部分复杂指令拆解，并按顺序执行。
5. 系统能够在 Canvas 画布上绘制基础图形、组合图案和完整场景。
6. 用户可以通过语音完成撤销、清空、保存等画布操作。
7. 项目尽量降低部署和调用成本，避免依赖复杂后端服务。

## 三、纯语音控制说明

本题要求用户不能使用鼠标或键盘完成绘图创作，仅通过语音指令完成绘图操作。

本项目最终版本采用纯语音控制方式实现。页面仅保留“启动语音控制”和“停止语音控制”两个辅助按钮，其中“启动语音控制”用于满足浏览器首次调用麦克风时必须由用户主动授权的安全限制。

用户完成麦克风授权后，绘图创作过程不再依赖键盘输入或鼠标绘制。用户可以通过语音完成以下操作：

- 绘制基础图形
- 绘制组合图案
- 随机生成完整风景画
- 执行复杂组合指令
- 撤销上一步
- 清空画布
- 保存图片
- 停止语音识别

因此，本项目的核心交互方式为：

语音输入 → 语音识别 → 指令拆解 → 指令解析 → Canvas 绘图

## 四、用户故事

### 用户故事 1：通过语音绘制基础图形

作为一名普通用户，我希望可以直接说出“在左边画一个红色圆形”，系统能够识别我的语音指令，并在画布对应位置绘制图形。

### 用户故事 2：通过语音控制图形颜色和位置

作为用户，我希望可以通过语音指定图形的颜色和位置，例如“在右边画一个蓝色矩形”，从而减少手动操作。

### 用户故事 3：通过语音生成完整画面

作为用户，我希望可以直接说“画一幅风景画”，系统能够自动生成包含天空、草地、太阳、云朵、房子、树木、小河等元素的完整画面。

### 用户故事 4：通过复杂语音指令完成多个步骤

作为用户，我希望可以一次说出“画一幅风景画，然后在左边画一个太阳，再在右边画一朵云”，系统能够拆解指令并依次执行。

### 用户故事 5：通过语音管理画布

作为用户，我希望可以通过“撤销”“清空画布”“保存图片”等语音指令管理绘图过程，提高操作容错性。

## 五、计划支持的指令能力

项目原计划支持以下指令能力：

1. 语音启动与语音识别  
   用户通过麦克风输入语音指令，系统自动识别语音内容。

2. 基础图形绘制  
   支持圆形、矩形、三角形等基础图形。

3. 颜色识别  
   支持红色、蓝色、绿色、黑色、黄色、紫色等基础颜色。

4. 位置识别  
   支持左边、右边、中间、上方、下方等位置表达。

5. 画布操作  
   支持撤销、清空画布、保存图片等操作。

6. 组合图案绘制  
   支持太阳、房子、云朵、星星、笑脸等组合图案。

7. 场景绘制  
   支持通过语音生成完整风景画。

8. 复杂指令拆解  
   支持将包含“然后”“再”“接着”等连接词的复杂指令拆解为多个子指令并顺序执行。

## 七、核心实现思路

本项目采用 HTML、CSS 和 JavaScript 开发。

HTML 负责页面结构，包括启动语音控制按钮、停止语音控制按钮、画布区域和状态提示区域。

CSS 负责页面样式，包括整体布局、按钮样式、画布边框和提示文字样式。

JavaScript 负责核心功能，包括语音识别、语音结果处理、复杂指令拆解、关键词解析、Canvas 绘图、撤销、清空和保存图片等。

系统流程如下：

1. 用户点击“启动语音控制”按钮。
2. 浏览器请求麦克风权限。
3. 用户授权后，系统进入连续语音监听状态。
4. 用户说出绘图指令。
5. 浏览器 Web Speech API 将语音转换为文本。
6. 系统根据连接词拆解复杂指令。
7. 系统逐条解析子指令中的关键词。
8. 系统判断图形、颜色、位置或操作类型。
9. 系统调用 Canvas 绘图函数完成绘制。
10. 用户可以继续通过语音进行后续绘图或画布操作。

## 八、指令解析逻辑

项目当前采用关键词规则解析方式。

例如用户说：

“在左边画一个红色圆形”

系统会解析出：

- “左” → 位置为 left
- “红” → 颜色为 red
- “圆” → 图形为 circle

然后调用画圆函数，在左侧坐标位置绘制红色圆形。

再例如用户说：

“画一幅风景画，然后在左边画一个太阳，再在右边画一朵云”

系统会先拆解为：

1. 画一幅风景画
2. 在左边画一个太阳
3. 在右边画一朵云

然后依次执行：

1. 调用随机风景画生成函数
2. 调用太阳绘制函数并定位到左边
3. 调用云朵绘制函数并定位到右边

## 九、复杂指令拆解设计

为了满足题目中“复杂指令的拆解与执行能力”的要求，项目设计了基于规则的复杂指令拆解方式。

当前支持根据以下词语进行拆解：

- 然后
- 再
- 接着
- 并且
- 同时
- 中文逗号
- 句号
- 分号

例如：

“清空画布，然后画一幅风景画，再在上方画一颗星星”

会被拆解为：

1. 清空画布
2. 画一幅风景画
3. 在上方画一颗星星

这种方式虽然不依赖大语言模型，但可以覆盖一部分常见的连续语音绘图表达，便于在短周期内实现稳定 demo。

## 十、语音识别准确性与容错性考虑

项目当前主要通过关键词匹配方式提升指令识别的稳定性。

例如：

- “矩形”“方形”“方块”都可以识别为矩形
- “云”“云朵”都可以识别为云朵
- “房子”“房屋”都可以识别为房子
- “撤销”“返回上一步”都可以触发撤销功能
- “停止语音”“停止识别”“结束语音”都可以停止语音控制

这种设计可以在一定程度上提高语音识别结果存在轻微差异时的容错能力。

目前暂未接入大语言模型，因此对于完全自由表达、长句语义理解和复杂同义词识别仍有局限。

## 十一、响应延迟考虑

项目采用前端静态页面实现，不依赖后端服务器或远程大模型接口，因此在语音识别完成后，指令解析和 Canvas 绘图都在浏览器本地完成。

主要响应流程为：

语音识别完成 → 文本解析 → Canvas 绘图

其中，绘图部分由本地 JavaScript 直接执行，响应速度较快。整体延迟主要来自浏览器语音识别过程本身。

为了降低响应延迟，本项目采用以下方式：

1. 不调用远程大模型接口
2. 不使用后端服务器
3. 使用规则解析代替复杂语义推理
4. 使用 Canvas API 直接绘图
5. 对复杂指令进行简单拆解后顺序执行

## 十二、随机风景画生成设计

为了提升作品完整度和展示效果，项目实现了随机风景画生成能力。

用户说：

“画一幅风景画”

系统会自动绘制一幅包含多个元素的完整画面，包括：

- 天空
- 草地
- 太阳
- 云朵
- 远山
- 房子
- 树木
- 小河
- 小路
- 花朵

同时，系统会对部分元素进行随机化处理，例如：

- 天空颜色随机
- 草地颜色随机
- 太阳位置随机
- 云朵数量随机
- 远山高度和颜色随机
- 房子位置随机
- 树木位置随机
- 小河是否出现随机
- 小路是否出现随机
- 花朵颜色和位置随机

因此，每次说“画一幅风景画”时，生成的画面都会有一定差异。

## 十三、成本控制策略

本项目采用浏览器原生能力实现，不依赖后端服务器、数据库或付费大模型接口。

主要成本控制方式包括：

1. 使用前端静态页面，无需服务器部署
2. 使用浏览器 Web Speech API 完成语音识别
3. 使用关键词规则解析指令，避免调用大模型接口
4. 使用 Canvas 完成绘图，降低系统复杂度
5. 所有功能均可在本地浏览器运行
6. 不需要额外购买云服务或模型服务

## 十四、未完成功能及原因

由于开发周期较短，目前暂未实现以下功能：

### 1. 任意复杂自然语言理解

当前主要采用关键词规则进行指令解析，能够支持基础图形、组合图案、随机风景画和部分复杂指令拆解，但暂不支持完全开放式的自然语言绘图。

原因：完全开放式自然语言理解通常需要接入大语言模型或更复杂的语义解析逻辑，短时间内实现成本较高。

### 2. 多轮精细编辑

当前支持撤销、清空和重新生成风景画，但暂不支持“把刚才的圆形变大”“把房子移动到右边一点”等精细化对象编辑。

原因：这需要系统记录每个图形对象的独立属性，并支持对象级别编辑，目前项目主要记录整体画布状态。

### 3. 更多图形和风格

当前实现了基础图形、组合图案和风景画场景，后续可以扩展人物、动物、交通工具、不同画风等内容。

原因：受开发时间限制，目前优先实现最能展示语音绘图流程的功能。

### 4. 完全免点击启动

由于浏览器安全限制，网页首次调用麦克风必须由用户主动触发授权，因此需要点击一次“启动语音控制”。

原因：这是浏览器安全策略要求，并非项目逻辑限制。进入语音控制模式后，绘图过程通过语音完成。

### 5. 更强的语音容错

当前支持部分同义词和关键词识别，但对于语音识别错误、长句误识别、口音差异等情况仍有优化空间。

原因：当前主要依赖浏览器语音识别结果和规则解析，暂未接入更高级的语义纠错模块。

## 十五、后续优化方向

后续可以从以下方向继续优化：

1. 接入大语言模型，提高复杂指令理解能力
2. 增加更多绘图元素，例如人物、动物、交通工具等
3. 支持多轮对话式修改图形
4. 支持对已绘制对象进行放大、缩小、移动和删除
5. 增加更多随机场景，例如夜空、海边、城市等
6. 优化页面交互和视觉效果
7. 提升语音识别失败时的容错提示
8. 增加指令执行日志，便于用户查看系统执行过程
9. 支持导出更多图片格式
10. 增加本地作品历史记录

## 十六、总结

VoiceDraw 是一个基于语音指令的轻量级 AI 绘图工具原型。项目实现了纯语音控制、连续语音识别、复杂指令拆解、Canvas 绘图、随机风景画生成、撤销、清空和保存等核心功能。

在实现策略上，项目采用浏览器原生 Web Speech API 和 Canvas API，避免复杂后端和高成本模型调用，适合在短周期内完成稳定的语音绘图 demo。

虽然当前版本仍无法完全理解任意复杂自然语言指令，但已经能够较完整地展示“语音控制绘图创作”的核心流程，并为后续接入大语言模型和对象级编辑功能提供了扩展基础。
