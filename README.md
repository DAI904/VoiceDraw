# VoiceDraw AI 纯语音绘图助手

## 一、项目简介

VoiceDraw 是一个基于浏览器语音识别能力实现的纯语音控制绘图工具。用户在启动语音控制并授权麦克风后，可以通过语音指令完成绘图创作、撤销、清空画布、保存图片等操作。

本项目选择题目二“AI 语音绘图工具”，重点实现“语音输入 → 指令拆解 → 指令解析 → Canvas 绘图”的核心流程。

由于浏览器安全限制，网页首次调用麦克风必须由用户主动触发授权，因此页面保留“启动语音控制”按钮。进入语音控制模式后，绘图创作过程不依赖键盘输入或鼠标绘制，主要通过语音指令完成。

## 二、已实现功能

目前项目已实现以下功能：

- 支持纯语音控制绘图
- 支持连续语音识别
- 支持复杂语音指令拆解
- 支持绘制圆形、矩形、三角形
- 支持识别红色、蓝色、绿色、黑色、黄色、紫色、橙色、粉色
- 支持识别左边、右边、中间、上方、下方位置
- 支持绘制太阳、房子、笑脸、云朵、星星等组合图案
- 支持随机生成完整风景画
- 支持语音撤销上一步
- 支持语音清空画布
- 支持语音保存图片
- 支持语音停止识别
- 支持将画布保存为 PNG 图片

## 三、使用方法

1. 使用 Chrome 或 Edge 浏览器打开 `index.html`。
2. 点击“启动语音控制”按钮，并允许浏览器使用麦克风。
3. 授权后，用户可以直接说出绘图指令，例如“画一幅风景画”。
4. 系统会自动识别语音内容，并根据指令完成绘图。
5. 用户可以继续通过语音说出“撤销”“清空画布”“保存图片”等操作指令。
6. 如果需要结束语音监听，可以说“停止语音控制”，也可以点击“停止语音控制”按钮。

说明：由于浏览器安全策略限制，首次启动麦克风需要用户主动点击按钮授权；授权后，绘图创作和画布操作均通过语音指令完成。

## 四、支持的语音指令示例

### 1. 基础绘图指令

- 在左边画一个红色圆形
- 在右边画一个蓝色矩形
- 在中间画一个黄色三角形
- 在上方画一个紫色圆形
- 在下方画一个绿色矩形

### 2. 组合图案指令

- 画一个太阳
- 在中间画一座房子
- 在右边画一朵云
- 在上方画一颗星星
- 画一个笑脸

### 3. 场景生成指令

- 画一幅风景画
- 重新画一幅风景画
- 画一幅完整画
- 生成一幅风景场景

### 4. 复杂指令示例

- 画一幅风景画，然后在左边画一个太阳，再在右边画一朵云
- 清空画布，然后画一幅风景画
- 画一个太阳，然后在右边画一座房子
- 画一幅风景画，接着在上方画一颗星星

### 5. 画布操作指令

- 撤销
- 返回上一步
- 清空画布
- 保存图片
- 停止语音控制
- 停止识别

## 五、核心实现思路

系统整体流程如下：

1. 用户点击“启动语音控制”按钮，完成麦克风授权。
2. 浏览器通过 Web Speech API 获取用户语音。
3. 系统将语音识别结果转换为文本。
4. JavaScript 根据“然后”“再”“接着”“并且”等连接词拆解复杂指令。
5. 系统对每条子指令进行关键词解析。
6. 根据指令内容判断图形、颜色、位置和操作类型。
7. 调用 Canvas API 完成对应绘图或画布操作。

例如用户说：

“画一幅风景画，然后在左边画一个太阳，再在右边画一朵云”

系统会拆解为：

1. 画一幅风景画
2. 在左边画一个太阳
3. 在右边画一朵云

然后依次执行绘图操作。

## 十、项目亮点

1. 纯语音控制  
   用户启动语音控制后，可以通过语音完成绘图、撤销、清空和保存等操作。

2. 复杂指令拆解  
   支持将包含“然后”“再”“接着”等连接词的复杂语音指令拆解为多个步骤依次执行。

3. 随机风景画生成  
   用户说“画一幅风景画”后，系统会随机生成由天空、草地、太阳、云朵、远山、房子、树木、小河、花朵等元素组成的完整画面。

4. 低成本实现  
   项目基于浏览器原生能力实现，不依赖后端服务器、数据库或付费大模型接口。

5. 可解释的规则解析  
   当前采用关键词规则解析方式，逻辑清晰，便于调试和扩展。

## 十一、未完成功能及原因

由于开发周期较短，目前暂未实现以下功能：

1. 任意复杂自然语言理解  
   当前主要采用关键词规则进行指令解析，能够支持基础图形、组合图案、随机风景画和部分复杂指令拆解，但暂不支持完全开放式的自然语言绘图。

2. 多轮精细编辑  
   当前支持撤销、清空和重新生成风景画，但暂不支持“把刚才的圆形变大”“把房子移动到右边一点”等精细化对象编辑。

3. 更多图形和风格  
   当前实现了基础图形、组合图案和风景画场景，后续可以扩展人物、动物、交通工具、不同画风等内容。

4. 完全免点击启动  
   由于浏览器安全限制，网页首次调用麦克风必须由用户主动触发授权，因此需要点击一次“启动语音控制”。进入语音控制模式后，绘图过程通过语音完成。

5. 更强的语音容错  
   当前已支持部分关键词识别，但对于语音识别错误、同音词、模糊表达等情况仍有优化空间。

## 十二、成本控制策略

本项目采用轻量级前端实现方式，不依赖后端服务器、数据库或付费大模型接口。

主要成本控制方式包括：

- 使用静态网页实现，无需服务器部署
- 使用浏览器 Web Speech API 完成语音识别
- 使用关键词规则解析指令，减少外部模型调用成本
- 使用 Canvas API 完成图形绘制
- 项目可在本地浏览器直接运行
